Caso de Estudio 1: Planta de Energía Nuclear

Este escenario te sumerge en el monitoreo crítico de una planta de energía nuclear, donde la detección temprana de anomalías es vital.

Descripción Detallada

En este caso, trabajarás con datos de sensores de temperatura de una planta de energía nuclear. El objetivo principal es aprender a construir consultas en el Lenguaje de Procesamiento de Eventos (EPL) que te permitan identificar situaciones en tiempo real. Esto incluye generar notificaciones cuando se detecten 3 tipos de eventos complejos:

  • Monitor: Monitorear la temperatura promedio cada X segundos.
  • Warning: Identificar si tenemos 2 temperaturas consecutivas por encima de un determinado umbral.
  • Critical: Alertarnos si tenemos 4 eventos consecutivos, con el primero por encima de un cierto umbral, y cada uno posterior mayor que el último, y el último siendo 1,5 veces mayor que el primero.

Este caso está diseñado para principiantes, proporcionando una guía paso a paso para familiarizarte con la sintaxis y la lógica de EPL. Tu misión será prevenir incidentes detectando patrones de eventos que, de otra forma, pasarían desapercibidos en un flujo constante de datos.

Pasos a seguir

Definir el tipo de evento

  • Cada lectura de temperatura se representa como un evento
  • Uso del esquema SCHEMA
  • Un evento debe definirse con: campos, tipos de datos
  • Ejemplo: create schema EventoSimple(objeto String, valor Integer);

Asociar nombres a las consultas

  • Identificar los diferentes patrones
  • Uso de anotaciones @Name
  • Ejemplo: @Name('Patron1')

Extraer data de los eventos simples que llegan en tiempo real

  • Uso de seleccion o consulta SELECT
  • Filtrar eventos según condiciones, Agrupar datos en el tiempo, Detectar patrones, Calcular promedios, conteos, etc
  • Ejemplo: select * from EventoSimple;

Evento complejo MONITOR

Saber el promedio de temperatura cada cierto tiempo

  • Usar ventanas temporales para agrupar eventos cada cierto tiempo (ej. cada XX segundos).
  • Usar funciones de agregación como avg().

Evento complejo WARNING

Detectar si se registran dos temperaturas consecutivas por encima de un umbral

  • Introducir patrones de eventos con pattern.
  • Usar condiciones dentro del patrón.

Evento complejo CRITICAL

Detectar una secuencia de 4 eventos donde cada uno es mayor que el anterior, y el último es 1.5 veces mayor que el primero

  • Introducir patrones de eventos con pattern.
  • Uso de múltiples variables en el patrón (a1, a2, a3, a4).
  • Usar comparaciones entre los eventos (a2.valor > a1.valor).

Creación de un flujo de eventos

  • Enviar los resultados de una consulta o patrón a un nuevo flujo de eventos
  • Uso del INSERT INTO
  • Ejemplo: insert into NuevoEvento select * from EventoSimple where valor > 100;
