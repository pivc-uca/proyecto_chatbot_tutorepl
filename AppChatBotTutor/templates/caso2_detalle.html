Caso de Estudio 2: Tráfico de Carreteras

En este escenario, tenemos un monitoreo inteligente del coche, cada sensor emite una señal cada pocos milisegundos.

Descripción Detallada

En este caso, trabajarás con los datos de los sensores de un coche que miden:

  • Presión de las ruedas.
  • Velocidad del coche.
  • Asiento ocupado o libre.
El objetivo principal es aprender a construir consultas en el Lenguaje de Procesamiento de Eventos (EPL) que te permitan identificar situaciones en tiempo real. Esto incluye generar notificaciones cuando se detecten los siguientes patrones de eventos:

  • ExplosionRueda (E): la presión de la rueda baja a <= 60 % de la presión original en menos de T segundos.
  • Colision (C): la velocidad del coche superior a 50 km/h pasa a 0 km/h en menos de T segundos.
  • OcupanteAbandonaAsiento (O): el asiento ocupado pasa a estar libre.
  • AccidenteOcupanteDespedido (A): se explota una rueda y, en los siguientes T segundos, se produce una colisión provocando que el ocupante abandone el asiento.

Para el evento complejo AccidenteOcupanteDespedido A se requerirá que se cumpla los tres (3) patrones simples de los eventos Explosión de rueda (E) → Colisión (C) → Ocupante abandono de asiento (O).

Este caso está diseñado para nivel intermedios, proporcionando una guía paso a paso para familiarizarte con la sintaxis y la lógica de EPL. Tu misión será buscar patrones simples o anomalías en tiempo real.

Pasos a seguir

Definir el tipo de evento

  • Cada sensor se representara dentro de la definicion del evento: presion de las ruedas (4), velocidad y asientos (4)
  • Uso del esquema SCHEMA
  • Un evento debe definirse con: campos , tipos de datos
  • Ejemplo: create schema Evento(objeto String, objeto2 Integer, objeto3 Boolean);

Asociar nombres a las consultas

  • Identificar los diferentes patrones
  • Uso de anotaciones @Name
  • Ejemplo: @Name('Patron1')

Extraer data de los eventos simples que llegan en tiempo real

  • Uso de seleccion o consulta SELECT
  • Filtrar eventos según condiciones, Agrupar datos en el tiempo, Detectar patrones, Calcular promedios, conteos, etc
  • Ejemplo: select * from Evento;

Patron simple Explosion Rueda

Detectar la caida de la presión de una rueda ( 60% ) muy brusca en muy poco tiempo

  • Usar operador de patron pattern con disparador every con seguimiento -> (followed-by) que significa "seguido por".
  • Para filtar usar operadores de comparacion como = y/0 <.
  • Usar filtro guard timer:within como condicionamiento que obliga a que un patrón ocurra dentro de un tiempo específico.

Patron simple Colision

Detectar que la velocidad del coche pasa de un valor mayor de 50 km/h a 0 km/h en un periodo corto

  • Usar operador de patron pattern con disparador every con seguimiento -> (followed-by) que significa "seguido por".
  • Para filtar usar operadores de comparacion como = y/0 <.
  • Usar filtro guard timer:within como condicionamiento que obliga a que un patrón ocurra dentro de un tiempo específico.

Patron simple Ocupante Abandona Asiento

Detectar que un asiento que estaba ocupado pasa a estar libre

  • Usar operador de patron pattern con disparador every con seguimiento -> (followed-by) que significa "seguido por".
  • Para filtar usar operadores de comparacion como =.

Patron complejo Accidente Ocupante Despedido

Este patrón encadena los tres eventos anteriores, Explosión de rueda (E) → Colisión (C) → Ocupante abandono de asiento (O) todo en X segundos, cada uno con una condición distinta

  • Usar operador de patron pattern con disparador every con seguimiento -> (followed-by) que significa "seguido por".
  • Usar filtro guard timer:within como condicionamiento que obliga a que un patrón ocurra dentro de un tiempo específico.

Creación de un flujo de eventos

  • Enviar los resultados de una consulta o patrón a un nuevo flujo de eventos
  • Uso de INSERT INTO
  • Ejemplo: insert into NuevoEvento select * from EventoSimple where valor > 100;
